네이밍 규칙

class명

  • 기본적으로 Block, Element, Modifer 규칙 이용
  • Block : 독립된 컴포넌트
  • Element : Block에서 분리되지 않는 블록의 일부분으로, 두 개의 밑줄(underscore, __)로 표현
    예) header-logo
  • Modifier : block 또는 element 의 변경된 상태를 정의하며, 두 개의 하이픈(--)으로 표현
    예) input-text

  • 소문자만 사용
  • 단어의 조합은 한개의 하이픈으로 사용(카멜표기법X)
    예) input-text
  • 숫자를 사용할 경우 02, 03으로 확장 고려
  • id는 공통 레이아웃에서만 사용

style

  • Block 안에서 scss 문법 사용 - 해당 부분 쉽게 옮길수 있도록
  • 반응형은 선택자 우선순위 문제가 있으므로, 랩핑을 최소화하고 선택자는 기본 1개, 부득이한 경우 최대 3개를 넘어가지 않도록 함
  • 선택자로 id 가 아닌 class만 사용
  • importnt 사용 지양
  • 이미지명은 한개의 언더바로 사용
  • 무분별한 스타일 중복 금지, 필요한 속성만 재정의

이미지

  • 소문자만 사용
  • svg 코드로 관리, 또는 svg 파일 사용
  • 파일명은 한개의 언더바로 연결하여 사용

기타

  • vs code 확장 프로그램 beautify 설치하여 동일한 포캣으로 파일 관리
  • 들여쓰기 공백 2 사용
  • 인코딩 UTF-8
